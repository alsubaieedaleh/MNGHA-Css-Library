1. Foundations

Foundations are the base layer of the CSS system. They should be used before components because components depend on tokens, base styles, layout, themes, and direction support.

Foundation Layers

  • Colors: brand, gray, error, warning, success, and semantic theme colors.
  • Typography: text sizes, line heights, and font weights.
  • Spacing: shared spacing variables used by components.
  • Radius: shared border radius values.
  • Layout: containers and grid utilities.
  • Direction: logical CSS support for LTR and RTL.
  • Themes: external and internal theme mapping using data-theme.

Example Usage

Import the compiled CSS file once in the page. 

<link rel="stylesheet" href="./../mngha-ui.css">

2. Theme System

Theme colors are controlled from the root HTML element. The same component can change its semantic colors when the theme changes.

Colors

  • External theme: data-theme="external"
  • Internal theme: data-theme="internal"
  • Theme primary color: used by primary buttons and selected states.
  • Theme light color: used by focus rings and light backgrounds.

Types

  • External
  • Internal

Example

HTML Code

<html lang="en" dir="ltr" data-theme="external">

<button class="mngha-button mngha-button--primary mngha-button--md" type="button">
  Primary Button
</button>

3. Header Component

The header is used for brand identity, main navigation, dropdown navigation, actions, and mobile menu entry.

Colors

  • Header background: component background color.
  • Header text: gray/semantic text color.
  • Active link: primary/brand color.
  • Dropdown menu: white/background surface with text and hover states.

Types

  • Top header meta area
  • Main header
  • Brand area
  • Navigation link
  • Navigation dropdown
  • Header actions
  • Mobile action

Example

Header is shown at the top of this page

Use the page header as the live component example. The Components item opens a dropdown menu using the same header classes.

HTML Code

<header class="mngha-header">
  <div class="mngha-header-main">
    <div class="mngha-container-fluid mngha-header-main-content">
      <a href="#" class="mngha-header-brand">
        <img class="mngha-header-logo" src="../assets/mngha-logo.webp" alt="Logo">
        <span class="mngha-header-title">System Name</span>
      </a>

      <nav class="mngha-header-nav" aria-label="Main navigation">
        <a href="#" class="mngha-header-link">Home</a>
        <a href="#" class="mngha-header-link">Services</a>
      </nav>
    </div>
  </div>
</header>

4. Button Component

Buttons are used for main actions, secondary actions, destructive actions, and status actions.

Colors

  • Primary: uses the current theme primary color.
  • Secondary: neutral secondary action.
  • Outline: bordered action.
  • Ghost: low-emphasis action.
  • Danger: error/destructive action.
  • Warning: warning action.
  • Success: success action.
  • Info: information action.

Types

  • Variants: primary, secondary, outline, ghost, danger, warning, success, info.
  • Sizes: small, medium, large.
  • States: default, hover, focus, disabled.

Example

HTML Code

<button class="mngha-button mngha-button--primary mngha-button--md" type="button">
  Primary Button
</button>

<button class="mngha-button mngha-button--secondary mngha-button--md" type="button">
  Secondary Button
</button>

<button class="mngha-button mngha-button--primary mngha-button--md" type="button" disabled>
  Disabled Button
</button>

5. Layout Grid

Layout utilities help arrange documentation sections, cards, buttons, and form fields using the same grid classes.

Colors

  • The layout grid itself does not define visual colors.
  • Children inside the grid keep their own component colors.

Types

  • Container: mngha-container
  • Fluid container: mngha-container-fluid
  • Grid: mngha-grid
  • Three-column grid: mngha-grid-cols-3
  • Card grid: mngha-card-grid

Colors

  • Disabled: muted gray background and text.

Types

  • Disabled input
  • Error input

  • Error textarea

8. Select / Dropdown

The custom select component uses a button and listbox menu. This allows full control over the selected option style, removes the native browser blue selected effect, and supports a check icon for the selected option.

Colors

  • Default border: neutral gray.
  • Hover border: darker gray.
  • Focus: bottom emphasis without changing layout height.
  • Selected option: white background with check icon.
  • Error: error border and focus emphasis.
  • Disabled: muted gray background and text.

Types

  • Default custom select
  • Error custom select
  • Disabled custom select

Example

  • Select city
  • Riyadh
  • Jeddah
  • Dammam

Choose your city.

  • Select city
  • Riyadh
  • Jeddah
  • Dammam

Please select a city.

  • Select city
  • Riyadh
  • Jeddah
  • Dammam

HTML Code

<div class="mngha-field">
  <label class="mngha-label" id="city-label">City</label>

  <div class="mngha-custom-select" data-custom-select>
    <input type="hidden" name="city" value="jeddah">

    <button
      class="mngha-custom-select-trigger"
      type="button"
      aria-haspopup="listbox"
      aria-expanded="false"
      aria-labelledby="city-label"
    >
      <span data-custom-select-value>Jeddah</span>
      <span class="mngha-custom-select-arrow" aria-hidden="true">⌄</span>
    </button>

    <ul class="mngha-custom-select-menu" role="listbox" aria-labelledby="city-label">
      <li class="mngha-custom-select-option" role="option" data-value="">Select city</li>
      <li class="mngha-custom-select-option" role="option" data-value="riyadh">Riyadh</li>
      <li class="mngha-custom-select-option is-selected" role="option" data-value="jeddah" aria-selected="true">Jeddah</li>
      <li class="mngha-custom-select-option" role="option" data-value="dammam">Dammam</li>
    </ul>
  </div>

  <p class="mngha-helper-text">Choose your city.</p>
</div>

10. File Upload

File upload supports a simple browse button version and a dropzone version. The uploaded file item uses status, file name, remove button, and optional helper text.

Colors

  • Browse button: strong gray background and white text.
  • File item: neutral gray surface and border.
  • Success status: brand/primary background.
  • Error status: error background and border.
  • Dropzone: dashed neutral border and muted background.

Types

  • Simple file upload
  • Dropzone file upload
  • Success file item
  • Error file item
  • Loading file item

11. Direction Support

Direction support allows the same HTML and CSS to work in English and Arabic by using logical properties and direction utilities.

Colors

  • Direction utilities do not define colors.
  • Text keeps the same typography and color tokens in both LTR and RTL.

Types

  • Page direction: dir="ltr" or dir="rtl"
  • Text start alignment: text-start
  • Text end alignment: text-end
  • Forced LTR content: dir-ltr
  • Forced RTL content: dir-rtl

Example

This paragraph uses text start alignment.

This paragraph uses text end alignment.

English value inside any page direction: user@example.com

نص عربي داخل أي اتجاه للصفحة

HTML Code

<html lang="en" dir="ltr">

<p class="text-start">
  This paragraph uses text start alignment.
</p>

<p class="dir-rtl">
  نص عربي داخل أي اتجاه للصفحة
</p>
