Tables

Tables organize structured data into rows and columns. The mngha table component supports responsive horizontal scrolling, theme-aware header variants, selectable rows, links, tags, statuses, actions, and pagination.

View Components

Overview

Responsive by default

Place the table inside mngha-table-wrapper. Wide tables scroll horizontally instead of exceeding the page.

Theme-aware headers

Add one modifier class to the table to use the active theme's brand color or a neutral dark header.

Accessible structure

Use a caption or accessible label, column scopes, descriptive control labels, and native table elements.

Default Table

The default table uses a light neutral header. Always wrap the table with mngha-table-wrapper so it remains contained on smaller screens.

Header Variants

Header variants are applied to mngha-table, not to individual rows or header cells. This keeps the entire header consistent and allows theme tokens to control its colors.

Brand header

Uses the active theme's primary brand and primary action text tokens.

Dark header

Class Reference

Class Element Purpose
.mngha-table-wrapper Container Contains wide tables and enables horizontal scrolling.
.mngha-table table Applies the base table layout, typography, and tokens.
.mngha-table--header-brand table Uses the active theme's brand header colors.
.mngha-table--header-dark table Uses a neutral dark header with inverse text.
.mngha-table-checkbox-cell th or td Sets the fixed width and spacing for checkbox columns.
.mngha-table-checkbox input Styles the table selection checkbox.
.mngha-table-link a Styles a primary link inside a table cell.
.mngha-table-tag span Displays a compact neutral data label.
.mngha-table-status span Displays a compact status pill with an indicator dot.
.mngha-table-action-cell th or td Sets the fixed width and alignment of the action column.
.mngha-table-action a or button Styles an icon-only row action.
.is-selected tr Applies the selected-row background.
.mngha-table-row--selected tr Explicit component modifier for a selected row.
.mngha-table-container Container Groups a table wrapper with its footer.
.mngha-table-footer Container Visually connects pagination or controls below a table.

Header Variant Usage

Variant Class Recommended use
Default .mngha-table General data tables and documentation tables.
Brand mngha-table mngha-table--header-brand Prominent application tables that should follow the active external or internal theme.
Dark mngha-table mngha-table--header-dark High-contrast neutral tables where brand color is not needed.

Accessibility

Identify the table

Use a visible caption when useful, or provide a concise aria-label when the surrounding heading already supplies the visible title.

Define header scope

Add scope="col" to column headers. Use scope="row" when the first cell identifies its row.

Label icon controls

Give checkboxes and icon-only action controls descriptive accessible labels that identify the affected row.

Keep native semantics

Use real table, thead, tbody, tr, th, and td elements. Do not recreate a data table using generic div elements.

Do not rely on color

Status text must remain understandable without its dot or background color.

Preserve keyboard access

Use links and buttons for interactive cells so controls remain focusable and operable from the keyboard.

Usage Guidance

Do

Keep column labels short, align data consistently, place row actions at the end, and wrap every table for responsive overflow.

Avoid

Do not apply theme color classes directly to individual header rows or cells. Use the approved table header modifier classes.

Use selection intentionally

Add checkboxes only when the application supports bulk actions or explicit row selection.

Keep actions clear

Use one clear primary row action or open a menu when several actions are available.
