MNGHA Universal CSS

Footer

The footer provides documentation links, social media actions, accessibility tools, copyright information, page metadata, and official logos. Keep the real footer in one shared partial and load it at the end of every documentation page.

View Components

Overview

Shared partial

Store the complete production footer in partials/footer.html and update it in one place.

Responsive

Navigation, tools, legal content, and logos automatically rearrange for tablet and mobile layouts.

Accessible

Use semantic navigation, visible group headings, named icon links, and descriptive logo alternative text.

Navigation Group

Use one section for each related group of footer links. Connect the section to its visible heading with aria-labelledby.

Add the Real Footer to a Page

Add one empty footer host after the main page content, then include the shared footer loader once before the closing body tag.

1. Create the partial

Keep the approved footer markup in partials/footer.html.

2. Add the host

Add data-mngha-footer after the main element.

3. Load the script

Include assets/js/footer.js once.

HTML

Page implementation 

<body>
  <div id="docs-header"></div>

  <main class="mngha-main">
    <div class="mngha-container">
      <!-- Documentation page content -->
    </div>
  </main>

  <div data-mngha-footer></div>

  <script src="./assets/docs.js"></script>
  <script src="./assets/code-block-copy.js"></script>
  <script src="./assets/government-sections.js"></script>
  <script src="./assets/js/footer.js"></script>
</body>

Footer Class Reference

Class Purpose
.mngha-footer Footer root and base component styles.
.mngha-footer-container Constrains and pads the footer content.
.mngha-footer-main Arranges navigation and tool sections.
.mngha-footer-navigation Contains grouped footer navigation.
.mngha-footer-group Groups one heading with related links.
.mngha-footer-heading Footer section heading and divider.
.mngha-footer-list Vertical footer link list.
.mngha-footer-link Main footer navigation link.
.mngha-footer-tools Contains social and accessibility tools.
.mngha-footer-action Square icon-only action.
.mngha-footer-meta-links Secondary footer links.
.mngha-footer-bottom Legal information and official branding area.
.mngha-footer-branding Responsive official logo group.
.mngha-footer-logo Shared logo link or container.

Usage Guidance

Do

  • Keep one shared footer partial for all documentation pages.
  • Use only real documentation page links.
  • Keep social links, copyright, and official logos consistent.
  • Test desktop, tablet, mobile, LTR, and RTL layouts.

Do not

  • Do not copy the complete footer into every page.
  • Do not place the real footer inside the main element.
  • Do not remove accessible names from icon-only links.
  • Do not hide the official logos on mobile.
