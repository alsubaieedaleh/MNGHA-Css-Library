Header Structure

Header root

Use mngha-header as the positioned root for the main header, desktop dropdown menu, and responsive mobile menu.

Main content

mngha-header-main-content aligns the brand, desktop navigation, actions, and mobile trigger in one row.

Responsive menu

Desktop navigation is hidden at 768px and below while the mobile action and mngha-mobile-menu become available.

Brand

The brand is an anchor containing the logo and title. Give the link an accessible name that describes its destination.

JavaScript Behavior

CSS controls presentation, while JavaScript synchronizes is-open, hidden, and aria-expanded. Escape closes open menus and clicking outside closes the desktop dropdown.

Header behavior

JAVASCRIPT

document.addEventListener("DOMContentLoaded", function () {
  const header = document.querySelector("[data-header-demo]");

  if (!header) return;

  const dropdownItem = header.querySelector(
    "[data-header-dropdown-item]"
  );

  const dropdownToggle = header.querySelector(
    "[data-header-dropdown-toggle]"
  );

  const mobileToggle = header.querySelector(
    "[data-header-mobile-toggle]"
  );

  const mobileMenu = header.querySelector(
    "[data-header-mobile-menu]"
  );

  const submenuToggle = header.querySelector(
    "[data-header-submenu-toggle]"
  );

  const submenu = header.querySelector(
    "[data-header-submenu]"
  );

  const submenuArrow = header.querySelector(
    "[data-header-submenu-arrow]"
  );

  function closeDesktopMenu() {
    if (!dropdownItem || !dropdownToggle) return;

    dropdownItem.classList.remove("is-open");
    dropdownToggle.setAttribute("aria-expanded", "false");
  }

  function closeMobileMenu() {
    if (!mobileMenu || !mobileToggle) return;

    mobileMenu.classList.remove("is-open");
    mobileToggle.setAttribute("aria-expanded", "false");
    mobileToggle.setAttribute(
      "aria-label",
      "Open navigation menu"
    );
  }

  dropdownToggle?.addEventListener("click", function (event) {
    event.stopPropagation();

    const nextOpen =
      !dropdownItem.classList.contains("is-open");

    closeDesktopMenu();

    if (nextOpen) {
      dropdownItem.classList.add("is-open");
      dropdownToggle.setAttribute("aria-expanded", "true");
    }
  });

  mobileToggle?.addEventListener("click", function () {
    const nextOpen =
      !mobileMenu.classList.contains("is-open");

    mobileMenu.classList.toggle("is-open", nextOpen);
    mobileToggle.setAttribute(
      "aria-expanded",
      String(nextOpen)
    );

    mobileToggle.setAttribute(
      "aria-label",
      nextOpen
        ? "Close navigation menu"
        : "Open navigation menu"
    );
  });

  submenuToggle?.addEventListener("click", function () {
    const nextOpen =
      submenuToggle.getAttribute("aria-expanded") !== "true";

    submenuToggle.setAttribute(
      "aria-expanded",
      String(nextOpen)
    );

    submenu.hidden = !nextOpen;

    if (submenuArrow) {
      submenuArrow.style.transition = "transform 160ms ease";
      submenuArrow.style.transform =
        nextOpen ? "rotate(180deg)" : "rotate(0deg)";
    }
  });

  document.addEventListener("click", function (event) {
    if (
      dropdownItem &&
      !dropdownItem.contains(event.target)
    ) {
      closeDesktopMenu();
    }
  });

  document.addEventListener("keydown", function (event) {
    if (event.key !== "Escape") return;

    closeDesktopMenu();
    closeMobileMenu();
  });
});

Accessibility

Label navigation

Give each navigation element a concise aria-label when the page contains more than one navigation region.

Identify the current page

Add aria-current="page" to the active desktop and mobile link, not only a visual active class.

Use buttons for toggles

Dropdown and mobile menu triggers should be buttons because they change interface state rather than navigate.

Synchronize state

Keep aria-expanded synchronized with is-open and the hidden attribute whenever a menu changes state.

Support Escape

Allow keyboard users to close open dropdown and mobile menus with the Escape key.

Describe icon buttons

Provide aria-label text for icon-only controls and hide purely decorative SVG elements with aria-hidden.

Usage Guidance

Do

Keep the brand concise, use a small number of top-level links, place related links in labeled dropdown groups, and preserve the same information architecture on mobile.

Avoid

Do not use anchor elements as menu toggles, omit active page semantics, duplicate IDs, leave aria-expanded outdated, or remove the visible keyboard focus indicator.
