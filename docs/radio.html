Radio

Radio controls allow users to select exactly one option from a related group. The mngha radio component supports labels, descriptions, checked states, disabled states, validation errors, keyboard focus, and responsive form layouts.

View Components

Overview

One choice per group

Give related radio inputs the same name so the browser allows only one option to remain selected.

Native behavior

The component keeps native radio semantics while replacing only the visual appearance.

Optional descriptions

Add supporting descriptions when users need more context to understand the difference between options.

Basic Radio Group

Use a fieldset and legend to identify the group, then place each native radio input inside mngha-radio-option.

Class Reference

Class Element Purpose
.mngha-radio-group Container Arranges related radio options vertically.
.mngha-radio-option label Groups the native input with its visible text.
.mngha-radio-input input[type="radio"] Applies the custom radio appearance and states.
.mngha-radio-content Container Groups the option label and optional description.
.mngha-radio-label Text Applies the main option-label typography.
.mngha-radio-description Text Displays supporting information under an option.
.mngha-radio-group--error Field or group Applies the error color to all radio inputs in the group.
.mngha-field fieldset or container Groups the legend, radio group, and feedback.
.mngha-label legend or label Applies the field-label typography.
.mngha-helper-text Text Displays supporting field guidance.
.mngha-error-text Text Displays the validation message.
.mngha-form-grid Container Arranges multiple radio fields responsively.

Accessibility

Group with fieldset

Use fieldset and legend so assistive technologies announce the question before each radio option.

Share the same name

All radio inputs in one group must use the same name value so only one option can be selected.

Keep labels clickable

Wrap each input and its text inside a label so the complete option remains easy to activate.

Preserve keyboard focus

Do not remove the focus-visible outline. Keyboard users need a clear indication of the active radio control.

Connect validation

Associate the group with its error message using aria-describedby and keep the message visible in text.

Explain disabled options

When useful, provide a description explaining why an option is unavailable instead of relying on the disabled appearance.

Usage Guidance

Do

Use radio groups when all options can be shown at once and the user must choose exactly one.

Avoid

Do not use radios for multiple selections, unrelated choices, or a very long list that would be easier to scan in another control.

Set a default carefully

Preselect an option only when there is a safe and sensible default. Avoid defaults that may cause an unintended action.

Keep option text distinct

Use concise labels and descriptions that clearly explain how the options differ.
