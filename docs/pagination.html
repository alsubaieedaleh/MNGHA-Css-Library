Pagination

Pagination divides large result sets into manageable pages. The mngha pagination component supports numbered pages, previous and next controls, compact access to additional pages, responsive overflow, and integration with table footers.

View Components

Overview

Direct page access

Numbered items allow users to move directly to a visible page.

Sequential navigation

Previous and next controls move one page at a time and clearly indicate when navigation is unavailable.

Compact overflow

The more-pages menu keeps large page ranges compact without removing access to additional pages.

Anatomy

A pagination control combines a navigation container, directional controls, numbered page items, and an optional more-pages dropdown.

Part Class Purpose
Responsive wrapper .mngha-pagination-wrapper Prevents the component from exceeding narrow layouts.
Navigation .mngha-pagination Groups all pagination controls into one labelled region.
Page item .mngha-pagination-item Moves directly to a numbered page.
Directional control .mngha-pagination-arrow Moves to the previous or next page.
More-pages control .mngha-pagination-more Opens the menu of additional pages.

Common States

The current page must be unique. Disable only the directional control that cannot move further through the result set.

First page

The previous control is disabled and page 1 is current.

Middle page

Both directional controls are available.

Last page

The next control is disabled and the last page is current.

State Reference

State Class or attribute Usage
Default page .mngha-pagination-item A page that is available but not currently displayed.
Current page .is-active and aria-current="page" Identifies the single page currently being displayed.
Disabled control disabled Prevents previous or next navigation at the beginning or end.
Open dropdown .is-open Applied to .mngha-pagination-dropdown while the additional page menu is visible.
Selected menu item .is-selected Identifies the selected option inside the additional page menu.

Class Reference

Class Element Purpose
.mngha-pagination-wrapper Container Provides horizontal overflow protection.
.mngha-pagination nav Defines the main pagination navigation region.
.mngha-pagination-item button or a Styles a numbered page control.
.mngha-pagination-arrow button or a Styles the previous or next control.
.mngha-pagination-dropdown Container Groups the more-pages trigger and its menu.
.mngha-pagination-more button Opens or closes the additional pages menu.
.mngha-pagination-menu Container Displays page options not shown in the main sequence.
.mngha-pagination-menu-item button Represents one selectable page inside the menu.
.mngha-table-container Container Groups a paginated table and its connected footer.
.mngha-table-footer Container Places pagination directly below a table.

Accessibility

Label the navigation

Add a concise aria-label to distinguish this pagination from other navigation regions.

Mark the current page

Apply aria-current="page" to exactly one page control.

Name icon controls

Provide clear labels such as Previous page, Next page, and Show more pages.

Use native disabled state

Apply the native disabled attribute to buttons that cannot be used.

Synchronize the menu state

Keep aria-expanded synchronized with the dropdown's visible state.

Preserve keyboard access

Use native buttons or links and allow Escape to close the additional pages menu.

Usage Guidance

Use pagination for large sets

Use pagination when loading all results at once would reduce performance or make content difficult to scan.

Keep the current page visible

The active page should remain in the main sequence whenever possible instead of being hidden inside the dropdown.

Keep the sequence compact

Show only the page numbers users need for orientation and place distant pages inside the more-pages menu.

Do not use pagination for steps

Multi-step forms and guided processes need a stepper or progress pattern rather than content pagination.

Dropdown JavaScript

Page loading remains application-specific. This script only opens and closes the additional pages menu, updates its ARIA state, manages the selected menu option, closes on outside click, and closes on Escape.

JavaScript

Pagination dropdown JavaScript

function initPaginationDropdowns() {
  const dropdowns = document.querySelectorAll(
    ".mngha-pagination-dropdown"
  );

  dropdowns.forEach(function (dropdown) {
    const trigger = dropdown.querySelector(
      ".mngha-pagination-more"
    );

    const items = dropdown.querySelectorAll(
      ".mngha-pagination-menu-item"
    );

    if (!trigger) return;

    trigger.addEventListener("click", function (event) {
      event.stopPropagation();

      const shouldOpen = !dropdown.classList.contains("is-open");

      closePaginationDropdowns();

      if (shouldOpen) {
        dropdown.classList.add("is-open");
        trigger.setAttribute("aria-expanded", "true");
      }
    });

    items.forEach(function (item) {
      item.addEventListener("click", function () {
        items.forEach(function (menuItem) {
          menuItem.classList.remove("is-selected");
        });

        item.classList.add("is-selected");

        closePaginationDropdowns();
        trigger.focus();
      });
    });
  });

  document.addEventListener("click", closePaginationDropdowns);

  document.addEventListener("keydown", function (event) {
    if (event.key === "Escape") {
      closePaginationDropdowns();
    }
  });
}

function closePaginationDropdowns() {
  const openDropdowns = document.querySelectorAll(
    ".mngha-pagination-dropdown.is-open"
  );

  openDropdowns.forEach(function (dropdown) {
    dropdown.classList.remove("is-open");

    const trigger = dropdown.querySelector(
      ".mngha-pagination-more"
    );

    if (trigger) {
      trigger.setAttribute("aria-expanded", "false");
    }
  });
}

initPaginationDropdowns();
