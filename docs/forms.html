Form Controls

Form controls collect text-based information using consistent labels, inputs, textareas, helper messages, validation feedback, responsive grids, and accessible field grouping.

View Components

Overview

Field structure

Use mngha-field to arrange a label, form control, and supporting or validation message as one logical field.

Shared states

Inputs and textareas support default, filled, focus, readonly, disabled, placeholder, and error states.

Responsive layout

Use mngha-form-grid for three-column forms that collapse to one column on small screens.

Basic Input

Place each input inside mngha-field. Connect the visible label using matching for and id values.

Example

Use the name shown on your official documents.

HTML

Basic input

<div class="mngha-field">
  <label class="mngha-label" for="full-name">
    Full name
  </label>

  <input
    class="mngha-input"
    id="full-name"
    name="full-name"
    type="text"
    placeholder="Enter your full name"
    aria-describedby="full-name-help"
  >

  <p class="mngha-helper-text" id="full-name-help">
    Use the name shown on your official documents.
  </p>
</div>

Validation Pattern

Apply the error class

Add mngha-input--error or mngha-textarea--error to the invalid control.

Expose the invalid state

Set aria-invalid to true after validation determines that the current value is invalid.

Connect the message

Give the error message an id and reference it from the control using aria-describedby.

Class Reference

Class Element Purpose
.mngha-field div or fieldset Arranges a label, control, and feedback vertically.
.mngha-label label or legend Applies the standard form-label typography.
.mngha-input input Applies the base single-line form-control appearance.
.mngha-input--error input Applies the input error border and focus indicator.
.mngha-textarea textarea Applies the base multi-line form-control appearance.
.mngha-textarea--error textarea Applies the textarea error border and focus indicator.
.mngha-helper-text Text Displays guidance or supporting information.
.mngha-error-text Text Displays validation feedback using the error color.
.mngha-form-grid Container Creates the responsive three-column form layout.

Accessibility

Use visible labels

Every input and textarea should have a clear visible label connected with matching for and id values.

Do not rely on placeholders

Placeholder text may provide an example, but it must not replace the field label.

Describe feedback

Connect helper and error messages with aria-describedby so they are announced with the control.

Use native attributes

Use required, disabled, readonly, autocomplete, and suitable input types instead of recreating their behavior.

Group related fields

Use fieldset and legend when multiple controls belong to the same named section or question.

Preserve focus behavior

Keep the component focus state visible for keyboard users and do not remove it with page-specific CSS.

Usage Guidance

Do

Use concise labels, appropriate native input types, unique IDs, connected helper messages, and clear validation feedback.

Avoid

Do not use placeholders as labels, mark readonly fields as disabled, display errors without explanation, or place unrelated fields inside one fieldset.
