Direction

Direction utilities help pages support both Arabic and English layouts. Use root direction, logical CSS properties, and direction helper classes to make components work correctly in RTL and LTR.

Root Direction

The recommended way to control page direction is from the root html element. Use rtl for Arabic pages and ltr for English pages.

Arabic page direction

<html lang="ar" dir="rtl" data-theme="external">

English page direction

<html lang="en" dir="ltr" data-theme="external">

Direction Utilities

Use direction utility classes when only one part of the page needs a specific direction.

Class Purpose Usage
.dir-rtl Forces right-to-left direction. Use for Arabic blocks inside an English page.
.dir-ltr Forces left-to-right direction. Use for English blocks, numbers, or technical content inside an Arabic page.
.text-start Aligns text to the start side. Left in LTR, right in RTL.
.text-end Aligns text to the end side. Right in LTR, left in RTL.

Direction utilities example

<div class="dir-rtl">
  محتوى عربي باتجاه من اليمين إلى اليسار
</div>

<div class="dir-ltr">
  English content from left to right
</div>

<p class="text-start">Start aligned text</p>
<p class="text-end">End aligned text</p>

Logical CSS Rules

Components should use logical properties so the same CSS works in Arabic and English without separate layout overrides.

Avoid Use Instead Reason
margin-left margin-inline-start Start side changes automatically between LTR and RTL.
margin-right margin-inline-end End side changes automatically between LTR and RTL.
padding-left padding-inline-start Padding follows the writing direction.
padding-right padding-inline-end Padding follows the writing direction.
text-align: left text-align: start Text aligns correctly in both Arabic and English.
text-align: right text-align: end Text aligns correctly in both Arabic and English.

Logical CSS example

.example {
  padding-inline-start: var(--spacing-md);
  padding-inline-end: var(--spacing-md);
  margin-inline-start: var(--spacing-lg);
  text-align: start;
}

Direction Usage Rules

Set direction once

Use the html dir attribute for the main page direction instead of setting direction on every component.

Avoid left and right

Do not use left and right spacing in component CSS unless the position is intentionally fixed.

Use start and end

Use logical properties and text alignment utilities so components support Arabic and English.
