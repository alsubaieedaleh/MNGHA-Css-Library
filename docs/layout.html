Layout

Layout utilities help structure pages using containers, sections, paragraphs, and grids. Use these classes to keep page width, spacing, and responsive behavior consistent across all applications.

Page Structure Grid System

Layout Overview

Container

Controls page width and horizontal padding.

Section

Creates vertical rhythm between page blocks.

Grid

Creates column-based layouts for cards, forms, tables, and content groups.

Page Structure

A standard page should use a main element, then a container, then sections. This keeps the page consistent and prevents content from touching the screen edges.

Basic page layout

<main class="mngha-main">
  <div class="mngha-container">

    <section class="mngha-section">
      ...
    </section>

  </div>
</main>

Container Utilities

Containers control horizontal layout. Use the standard container for normal pages and the fluid container for full-width areas like headers or wide sections.

Class Purpose Behavior
.mngha-container Standard centered page container. Full width until max width, centered with responsive padding.
.mngha-container-fluid Full-width container. Uses full available width with responsive horizontal padding.
.mngha-paragraph Readable text width. Limits paragraph width so long text stays readable.

Container examples

<div class="mngha-container">
  Standard page content
</div>

<div class="mngha-container-fluid">
  Full-width content
</div>

<p class="mngha-paragraph">
  Long readable paragraph content.
</p>

Container Tokens

These tokens define the container width, page padding, and readable paragraph width.

Token Value Usage
--container-padding-mobile 1rem Horizontal padding on smaller screens.
--container-padding-desktop 2rem Horizontal padding from tablet size and above.
--container-max-width-desktop 80rem Maximum width of the standard page container.
--paragraph-max-width 45rem Maximum readable width for long paragraph text.

Section Layout

Use sections to divide page content into clear blocks. Sections should contain related content such as a heading, description, table, card group, or example.

Section example

<section class="mngha-section">
  <h2 class="mngha-section-title">Section title</h2>

  <p class="mngha-paragraph">
    Section description.
  </p>
</section>

Grid System

Grid utilities create consistent column layouts. Use fixed column grids when you control the content, and use the responsive grid when the layout should adapt across screen sizes.

Class Columns Recommended Usage
.mngha-grid Base grid wrapper Required before using fixed grid column classes.
.mngha-grid-cols-12 12 columns Advanced layouts and large page structures.
.mngha-grid-cols-6 6 columns Medium card groups and form sections.
.mngha-grid-cols-5 5 columns Special layouts that need five equal columns.
.mngha-grid-cols-3 3 columns Feature cards, overview cards, and small groups.
.mngha-grid-cols-2 2 columns Two-column content, side-by-side examples, and forms.
.mngha-grid-responsive 4 → 6 → 12 columns Responsive page layouts that adapt at breakpoints.

Layout Usage Rules

Use containers

Do not place page content directly under body. Wrap it with main and container classes.

Use sections

Divide pages into clear sections instead of using random margins between blocks.

Use system spacing

Do not create custom layout spacing. Use the ready layout utilities and spacing tokens.
