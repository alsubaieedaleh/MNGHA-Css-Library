File Upload

File upload controls allow users to select supporting documents and review loading, success, and validation states.

View Components

Overview

Simple upload

Use a compact browse control for regular forms.

Dropzone upload

Use a larger upload area when attaching files is a primary task.

File states

Show loading, success, and error feedback for each file.

Simple File Upload

Use the simple layout for a title, guidance, and compact browse control.

Example

Upload supporting document

Maximum file size is 2MB. Supported formats are PDF, JPG, and PNG.

HTML

Simple file upload

<div class="mngha-file-upload">
  <div class="mngha-file-upload-content">
    <p class="mngha-file-upload-title">Upload supporting document</p>
    <p
      class="mngha-file-upload-description"
      id="supporting-document-description"
    >
      Maximum file size is 2MB. Supported formats are PDF, JPG, and PNG.
    </p>
  </div>

  <input
    class="mngha-file-upload-input"
    id="supporting-document"
    type="file"
    accept=".pdf,.jpg,.jpeg,.png"
    aria-describedby="supporting-document-description"
  >

  <label
    class="mngha-file-upload-button"
    for="supporting-document"
  >
    Browse Files
  </label>
</div>

Basic JavaScript

CSS does not create file items automatically. This minimal example displays one selected file and allows it to be removed.

JavaScript

Render one selected file

document
  .querySelectorAll("[data-file-upload]")
  .forEach(function (upload) {
    const input = upload.querySelector(".mngha-file-upload-input");
    const list = upload.querySelector(".mngha-file-list");

    if (!input || !list) return;

    input.addEventListener("change", function () {
      list.replaceChildren();

      const file = input.files[0];
      if (!file) return;

      const item = document.createElement("div");
      item.className =
        "mngha-file-item mngha-file-item--success";

      const status = document.createElement("span");
      status.className = "mngha-file-status";
      status.setAttribute("aria-hidden", "true");
      status.textContent = "✓";

      const name = document.createElement("span");
      name.className = "mngha-file-name";
      name.textContent = file.name;

      const remove = document.createElement("button");
      remove.className = "mngha-file-remove";
      remove.type = "button";
      remove.textContent = "×";
      remove.setAttribute("aria-label", `Remove ${file.name}`);

      remove.addEventListener("click", function () {
        input.value = "";
        list.replaceChildren();
      });

      item.append(status, name, remove);
      list.append(item);
    });
  });

Accessibility

Connect labels and inputs

The label for value must match the native file input id.

Describe restrictions

State formats, size limits, and quantity limits before selection.

Announce updates

Use aria-live="polite" on the file list when JavaScript updates it.

Label removal controls

Include the affected file name in every remove button label.

Validate beyond accept

The accept attribute guides the picker but does not securely validate files.

Explain errors

Provide a text reason and corrective action instead of relying on color.

Usage Guidance

Do

Validate every file, show progress when needed, and allow removal before submission.

Avoid

Do not hide file restrictions or expose raw technical error messages.

Choose the right variant

Use simple upload in compact forms and dropzone for prominent upload tasks.

Validate securely

Check type, size, content, and permissions again on the server.
