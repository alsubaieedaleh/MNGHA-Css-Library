Layout Theme Direction

Foundations

Foundations are the base layer of mngha Universal CSS. This page documents the tokens and layout utilities currently available in the system: colors, typography, spacing, radius, container, grid, theme, and direction.

View Colors View Typography

Foundation Checklist

Colors

Core, external primary, secondary, gray, status, semantic aliases, internal colors, and text aliases.

Typography

Font family, defaults, weights, text sizes, display sizes, line heights, and letter-spacing tokens.

Spacing

Primitive spacing scale and semantic spacing aliases from none to 12xl.

Radius

Primitive and semantic radius tokens for controls, cards, pills, and badges.

Container

Centered container, fluid container, paragraph width, and responsive page padding.

Grid

Fixed column grid utilities and responsive 4/6/12-column grid behavior.

Color System

Use primitive color tokens to build aliases, and use semantic aliases inside components when the color has a meaning such as primary, success, warning, error, info, or text.

Main Color Rule

Use the available main tokens

Your current foundation files define --color-primary for the external main color and --color-internal-primary for the internal main color. The previous page used --color-brand-primary and --color-brand-light, but those aliases are not defined in the files you shared. 

/* Current external main color */
color: var(--color-primary);

/* Current internal main color */
color: var(--color-internal-primary);

Review note

Theme and Direction

Theme

The root element can store the current theme using data-theme. Your docs toggle already switches this value.

<html lang="en" dir="ltr" data-theme="external">
<html lang="en" dir="ltr" data-theme="internal">

Direction

Direction is controlled by the root dir attribute. Your CSS should keep using logical properties like margin-inline, padding-inline, and inset-inline.

<html lang="en" dir="ltr">
<html lang="ar" dir="rtl">

Foundation Rules

Use tokens

Do not add raw colors, font sizes, spacing, or radius values inside components.

Use semantic meaning

Use semantic aliases when the token represents meaning, such as success, warning, error, info, or text.

Use layout utilities

Use container and grid utilities for docs pages, templates, and component examples.
