Foundations

Foundations are the base layer of mngha Universal CSS. This page documents the tokens and layout utilities currently available in the system: colors, typography, spacing, radius, width, container, grid, theme, and direction.

Colors

Core, external primary, secondary, gray, status, semantic aliases, internal colors, and text aliases.

Typography

Font family, defaults, weights, text sizes, display sizes, line heights, and letter-spacing tokens.

Primitive spacing scale and semantic spacing aliases from none to 12xl.

Radius

Primitive and semantic radius tokens for controls, cards, pills, and badges.

Fixed column grid utilities and responsive 4/6/12-column grid behavior.

Main Color Rule

Use theme aliases for main colors

Components should use theme-aware aliases such as --color-brand-primary, --color-action-primary-bg, and --color-link. Primitive palettes like --color-primary-600 and --color-internal-primary should stay in the foundation layer.

Theme-aware color usage

/* Brand color */
color: var(--color-brand-primary);

/* Primary action */
background: var(--color-action-primary-bg);
color: var(--color-action-primary-text);
border-color: var(--color-action-primary-border);

/* Links */
color: var(--color-link);

Theme and Direction

Theme

The root element can store the current theme using data-theme. Your docs toggle already switches this value.

Theme attribute

<html lang="en" dir="ltr" data-theme="external">
<html lang="en" dir="ltr" data-theme="internal">

Direction

Direction is controlled by the root dir attribute. Your CSS should keep using logical properties like margin-inline, padding-inline, and inset-inline.

Direction attribute

<html lang="en" dir="ltr">
<html lang="ar" dir="rtl">

Foundation Rules

Use tokens

Do not add raw colors, font sizes, spacing, or radius values inside components.

Use semantic meaning

Use semantic aliases when the token represents meaning, such as success, warning, error, info, or text.

Use layout utilities

Use container and grid utilities for docs pages, templates, and component examples.
