Cards

Cards group related content and actions inside a contained visual surface. The mngha card component supports icons, titles, descriptions, actions, responsive grids, interactive states, selection, disabled states, checkboxes, and header controls.

View Components

Overview

Structured content

Combine a card with content, title, description, icon, header, and action classes according to the information it contains.

Responsive grid

The card grid displays three columns on larger screens and automatically changes to one column on small screens.

State support

Interactive, selected, and disabled modifiers communicate the current behavior and state of a card.

Basic Card

Use the base card with a content wrapper, title, and description for a simple contained content block.

Digital services

Manage the organization's digital services, platforms, and integration channels from one location.

Basic card

<article class="mngha-card">
  <div class="mngha-card-content">
    <h3 class="mngha-card-title">Digital services</h3>

    <p class="mngha-card-description">
      Manage the organization's digital services, platforms, and
      integration channels from one location.
    </p>
  </div>
</article>

Background Variants

Cards use a white background by default. Use the muted modifier for a neutral secondary surface or the brand-soft modifier for a light, theme-aware brand surface.

Default background

The standard card uses the white surface.

Muted background

Use for secondary information or visually grouped content.

Brand-soft background

Uses the active theme's light brand surface.

Interactive Card

Use mngha-card--interactive when the complete card opens a destination or performs one clear action. Do not place nested buttons or links inside a fully interactive card.

Use the selected modifier to communicate a current choice. Use the disabled modifier when a card is unavailable and ensure interactive controls inside it are also disabled.

Selected card

Disabled card

Card Toggle Control

Use mngha-card-toggle for a compact header control such as expand, collapse, show details, or hide details. JavaScript should update its expanded state and the related content.

Request details

The control can be connected to expandable content using aria-expanded and aria-controls.

Unavailable details

The toggle becomes unavailable when the related card is disabled.

HTML

Card toggle control

<article class="mngha-card">
  <div class="mngha-card-header">
    <div class="mngha-card-content">
      <h3 class="mngha-card-title">Request details</h3>
    </div>

    <button
      class="mngha-card-toggle"
      type="button"
      aria-expanded="false"
      aria-controls="request-details"
    >
      Show details
    </button>
  </div>

  <div id="request-details">
    <p class="mngha-card-description">
      Additional request information.
    </p>
  </div>
</article>

Class Reference

Class Element Purpose
.mngha-card Container Applies the card surface, border, radius, shadow, and spacing.
.mngha-card-grid Container Arranges cards in a responsive three-column grid.
.mngha-card-icon span or container Displays an icon inside a circular brand surface.
.mngha-card-content Container Groups a card title, description, and related content.
.mngha-card-title Heading Applies the card heading typography.
.mngha-card-description Text Applies the card supporting-text typography.
.mngha-card-actions Container Arranges card buttons and links in a wrapping action row.
.mngha-card--muted Card Uses a neutral secondary card background.
.mngha-card--brand-soft Card Uses the active theme's light brand card background.
.mngha-card--interactive Card Adds an interactive cursor and hover border behavior.
.mngha-card--selected Card Applies the selected-state border.
.mngha-card--disabled Card Applies disabled colors, removes the shadow, and changes the cursor.
.mngha-card-header Container Aligns heading content with a checkbox or toggle control.
.mngha-card-checkbox input[type="checkbox"] Styles a card selection checkbox using the brand color.
.mngha-card-toggle button Styles a compact text control inside the card header.

Accessibility

Use meaningful structure

Use article, section, list item, or another semantic container that matches the role of the card content.

Preserve heading order

Select heading levels according to the surrounding page structure rather than choosing a level only for appearance.

Avoid nested interaction

Do not place buttons or links inside a card when the complete card is already one interactive link or control.

Label controls

Give checkboxes and compact toggle controls clear accessible names describing the affected card or content.

Communicate selection

Use a checkbox, radio control, text, or another semantic state in addition to the selected border.

Disable real controls

A disabled card style does not disable nested controls by itself. Apply disabled or aria-disabled to the actual controls.

Usage Guidance

Do

Keep each card focused on one subject, use concise titles, and place related cards together inside the responsive card grid.

Avoid

Do not place unrelated content, too many actions, or long page sections inside a single card.

Use states consistently

Apply interactive, selected, and disabled modifiers only when the card behavior genuinely matches the state.

Keep actions predictable

Use a fully interactive card for one destination, or use separate action controls inside a non-interactive card.
