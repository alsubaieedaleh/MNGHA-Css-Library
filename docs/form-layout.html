Form Layouts

Form layout classes combine the existing MNGHA fields, inputs, textareas, selects, radio controls, and buttons into complete, structured, responsive, and accessible forms.

View Components

Overview

Complete structure

Use a form header, body, related sections, and an action area to create a predictable complete-form structure.

Responsive rows

Use one column by default and place only closely related fields inside the optional two- or three-column rows.

Validation support

Combine existing inline error states with the form-level error summary to help users find and correct invalid fields.

Basic Form Layout

Start with mngha-form. Add mngha-form--contained when the form needs its own bordered surface. Controls continue to use the existing form classes.

Example

Create department

Enter the basic department information.

Department details

HTML

Basic form layout

<form
  class="mngha-form mngha-form--contained"
  action="#"
  method="post"
>
  <header class="mngha-form-header">
    <h3 class="mngha-form-title">Create department</h3>

    <p class="mngha-form-description">
      Enter the basic department information.
    </p>
  </header>

  <div class="mngha-form-body">
    <section
      class="mngha-form-section"
      aria-labelledby="department-details-title"
    >
      <header class="mngha-form-section-header">
        <h4
          class="mngha-form-section-title"
          id="department-details-title"
        >
          Department details
        </h4>
      </header>

      <div class="mngha-form-row">
        <div class="mngha-field">
          <label class="mngha-label" for="department-name">
            Department name
            <span
              class="mngha-required-marker"
              aria-hidden="true"
            >*</span>
          </label>

          <input
            class="mngha-input"
            id="department-name"
            name="department-name"
            type="text"
            required
          >
        </div>
      </div>
    </section>
  </div>

  <footer class="mngha-form-actions">
    <button
      class="mngha-button mngha-button--primary mngha-button--md"
      type="submit"
    >
      Create department
    </button>

    <button
      class="mngha-button mngha-button--secondary mngha-button--md"
      type="reset"
    >
      Reset
    </button>
  </footer>
</form>

Responsive Form Rows

A basic mngha-form-row uses one column. Add mngha-form-row--2 or mngha-form-row--3 only when fields are closely related. All variants collapse to one column on small screens.

Example

<div class="mngha-form">

        First name
      </label>
      <input

        name="first-name"
        type="text"

      <label class="mngha-label" for="last-name">

        name="last-name"

  </div>

  <div class="mngha-form-row mngha-form-row--3">
    <div class="mngha-field">

Required and Optional Fields

Use mngha-required-marker for the required asterisk. Its color uses the design-system error-700 token. Use mngha-optional-text for optional fields.

Example

Optional

HTML

Required and optional labels

Validation Summary

When submission fails, place the error summary before the form body. Link every summary message to its invalid field and keep the same message beside that field.

Example

Contact support

There is a problem

  • Enter a valid email address.

Your details

Enter a valid email address.

HTML

Form with validation summary

<form
  class="mngha-form mngha-form--contained"
  action="#"
  method="post"
  novalidate
>
  <header class="mngha-form-header">
    <h3 class="mngha-form-title">Contact support</h3>
  </header>

  <div
    class="mngha-form-error-summary"
    role="alert"
    tabindex="-1"
  >
    <h4 class="mngha-form-error-summary-title">
      There is a problem
    </h4>

    <ul class="mngha-form-error-list">
      <li>
        <a href="#support-email">
          Enter a valid email address.
        </a>
      </li>
    </ul>
  </div>

  <div class="mngha-form-body">
    <section
      class="mngha-form-section"
      aria-labelledby="support-details-title"
    >
      <header class="mngha-form-section-header">
        <h4
          class="mngha-form-section-title"
          id="support-details-title"
        >
          Your details
        </h4>
      </header>

      <div class="mngha-form-row">
        <div class="mngha-field">
          <label class="mngha-label" for="support-email">
            Email address
            <span
              class="mngha-required-marker"
              aria-hidden="true"
            >*</span>
          </label>

          <input
            class="mngha-input mngha-input--error"
            id="support-email"
            name="support-email"
            type="email"
            aria-invalid="true"
            aria-describedby="support-email-error"
            required
          >

          <p class="mngha-error-text" id="support-email-error">
            Enter a valid email address.
          </p>
        </div>
      </div>
    </section>
  </div>

  <footer class="mngha-form-actions">
    <button
      class="mngha-button mngha-button--primary mngha-button--md"
      type="submit"
    >
      Submit again
    </button>
  </footer>
</form>

Compact Form

Add mngha-form--compact for short forms inside dialogs or side panels. Do not use the compact variant to force long forms into small containers.

Example

Rename item

HTML

Compact form

<form
  class="mngha-form mngha-form--contained mngha-form--compact"
  action="#"
  method="post"
>
  <header class="mngha-form-header">
    <h3 class="mngha-form-title">Rename item</h3>
  </header>

  <div class="mngha-form-body">
    <div class="mngha-form-row">
      <div class="mngha-field">
        <label class="mngha-label" for="item-name">Name</label>
        <input
          class="mngha-input"
          id="item-name"
          name="item-name"
          type="text"
        >
      </div>
    </div>
  </div>

  <footer class="mngha-form-actions mngha-form-actions--end">
    <button
      class="mngha-button mngha-button--primary mngha-button--md"
      type="submit"
    >
      Save
    </button>

    <button
      class="mngha-button mngha-button--secondary mngha-button--md"
      type="button"
    >
      Cancel
    </button>
  </footer>
</form>

Class Reference

Class Element Purpose
.mngha-form form or container Creates the base complete-form layout.
.mngha-form--contained Modifier Adds padding, border, background, and shadow.
.mngha-form--compact Modifier Reduces spacing for dialogs and side panels.
.mngha-form-header header Contains the form title and description.
.mngha-form-body Container Contains form sections and field rows.
.mngha-form-section section Groups related fields under one heading.
.mngha-form-row Container Creates a single-column field row.
.mngha-form-row--2 Modifier Creates a responsive two-column field row.
.mngha-form-row--3 Modifier Creates a responsive three-column field row.
.mngha-form-column--full Field modifier Makes a field span every column in its row.
.mngha-form-actions footer or container Places submit and secondary actions after the fields.
.mngha-required-marker span Displays the required asterisk using error-700.
.mngha-optional-text span Displays an optional-field label.
.mngha-form-error-summary Container Displays form-level validation errors.

Accessibility

Use visible labels

Every interactive control must have a visible label connected using matching for and id values.

Keep native required

The visual asterisk is hidden from assistive technology. Keep the native required attribute on the form control.

Use logical sections

Group only related fields and connect each section heading with aria-labelledby when the section needs an accessible name.

Connect error messages

Set aria-invalid and connect inline error text using aria-describedby after validation fails.

Focus the summary

Move keyboard focus to the error summary after an invalid form submission so the problem is announced immediately.

Preserve source order

Keep the visual and DOM order logical when responsive rows collapse from multiple columns to one column.

Usage Guidance

Do

Use one column by default, group related fields, keep actions at the end, use concise headings, and provide clear validation.

Avoid

Do not place unrelated fields in one row, use too many columns, hide required information, or place submit actions above fields.
